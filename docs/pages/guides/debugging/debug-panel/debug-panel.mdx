## 简介

APP调试面板主要用于在开发&测试阶段，为业务开发和测试提供便捷的调试测试能力，意在提升开发&测试效率

目前APP调试面板统一使用JS开发，磨平了之前iOS和Android两端在交互和功能上的差异性，对使用者来说更加友好

调试面板支持两种模式：

- Debug模式（Debug包）：用于开发阶段调试
- Release模式（Release包）：用于测试阶段回归功能

### 清除缓存

清除缓存数据这个功能，能够清理掉APP在使用过程中产生的缓存数据，避免在使用过程中因为缓存数据导致的异常

### 系统设置页

系统设置页按钮的功能是直接唤起手机系统页面，，便捷设置APP的系统相关功能

### 设置环境

设置环境页面支持自定义修改当前BaseUrl的环境，直接输入环境数字，点击保存并重启APP即可。切换成功后，APP内部也会同时从codePush服务请求对应环境的developmentKey，接受codepush。大大降低native APP的打包频率

### 动态调试bundle

动态调试bundle页面，支持用户手动输入IP地址，或者一键扫码绑定IP地址，需要调试哪一个bundle，直接勾选对应的开关，点击保存并重启APP按钮后，重启APP即可启动bundle服务，不需要重新构建native包，大大降低native APP的打包频率

### Reload bundle

ReloadBundle 按钮功能沿用react native DevMenu 中的Reload bundle 能力，点击Reload bundle 会重新加载当前bundle
